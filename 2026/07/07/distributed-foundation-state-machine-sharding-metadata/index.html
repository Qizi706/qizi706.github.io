以 BlockGroup 为例，它不仅是一组 KV Cache block 的集合，还是一个有生命周期、有位置、有 owner、有版本的分布式对象。系统不仅要知道它是什么状态，还要知道它在哪里、谁能修改它、客户端看到的位置是不是最新的。

这一篇的目标是：能把一个分布式对象的生命周期讲清楚，并理解一个请求如何找到它需要的 KV Cache。
